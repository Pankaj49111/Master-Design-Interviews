# External Resources

This folder contains external PDFs and reference books gathered to support interview preparation for system design and software engineering topics. The files are third-party resources (books, guides, and curated archives). Keep personal copies for study; follow each resource's license for redistribution.

## Purpose
- Collect foundational system design and architecture references in one place.
- Provide quick descriptions and a suggested reading order for interview prep.

## Files in this folder
Below are the files currently in this folder with short notes to help you pick what to read next.

- [01 OReilly Head First - Head First Design Patterns - Freeman](01%20OReilly%20Head%20First%20-%20Head%20First%20Design%20Patterns%20-%20Freeman.pdf)
  - A beginner-friendly, example-driven introduction to object-oriented design patterns. Good for learning classic design patterns and their intent.

- [05 Designing_Distributed_Systems](05%20Designing_Distributed_Systems.pdf)
  - Practical guidance on designing distributed systems, trade-offs, and anti-patterns.

- [06 Building Microservices](06%20Building%20Microservices.pdf)
  - Focuses on building and operating microservice architectures, including organizational and runtime concerns.

- [07 Microservice_Pattern](07%20Microservice_Pattern.pdf)
  - Pattern catalog for microservices including decomposition, integration, and reliability patterns.

- [ByteByteGo-Big-Archive-System-Design-2023](ByteByteGo-Big-Archive-System-Design-2023.pdf)
  - A large archive of system design notes and interview-style problems (2023 collection).

- [ByteByteGo-System Design Big Archive 2024](ByteByteGo-System%20Design%20Big%20Archive%202024.pdf)
  - Updated archive with more recent system design materials (2024 collection).

- [Code Simplicity - Max Kanat-Alexander](Code%20Simplicity%20-%20Max%20Kanat-Alexander.pdf)
  - Essays and guidance on writing simple, maintainable code and pragmatic software engineering practices.

- [Cracking the Coding Interview (189 problems)](Cracking_the_Coding_Interview_189_Programming_Questions_and_Solutions.pdf)
  - Classic coding interview book (algorithms and data structures practice). Useful for low-level design and algorithmic thinking.

- [Database Internals](Database%20Internals.pdf)
  - Deep-dive into database design, storage engines, indexing, and transactional systems — useful for backend and data-heavy system design.

- [Designing Data-Intensive Applications by Martin Kleppmann](Designing%20Data%20Intensive%20Applications%20by%20Martin%20Kleppmann.pdf)
  - Comprehensive book on data systems, scalability, consistency, and system design trade-offs.

- [Domain-Driven Design — Eric Evans](Domain-Driven%20Design_%20Tackling%20-%20Eric%20Evans_14.pdf)
  - Foundational text on domain modeling, bounded contexts, and modeling complexity in large systems.

- [Grokking the System Design Interview](Grokking-the-system-design-interviewpdf-5-pdf-free%20%281%29.pdf)
  - Grokking-style guide with common interview examples and suggested solutions.

- [Linux Pocket Guide — Essential Commands (Barrett)](Linux%20Pocket%20Guide%20Essential%20Commands%20by%20Daniel%20J.%20Barrett%20%28z-lib.org%29.pdf)
  - Quick-reference for Linux commands; handy for devops and infrastructure tasks.

- [System Design Interview — Alex Xu (Vol 1)](System%20Design%20Interview%20by%20Alex%20Xu.pdf)
  - Popular interview preparation book covering common system design problems and sample solutions.

- [System Design Interview — Alex Xu (Vol 2)](System%20Design%20Interview%20By%20Alex_Xu_Vol2.pdf)
  - Second volume with additional problems and expanded solutions.

## Recommended reading order (quick path)
1. Head First Design Patterns (patterns foundation)
2. Code Simplicity (coding & maintainability mindset)
3. Cracking the Coding Interview (algorithms practice)
4. Designing Data-Intensive Applications (data systems & trade-offs)
5. System Design Interview (Alex Xu) + Grokking resources (interview practice)
6. Designing Distributed Systems / Database Internals / Microservice patterns (deeper architecture topics)
7. ByteByteGo archives (browse problems and real interview discussions)
